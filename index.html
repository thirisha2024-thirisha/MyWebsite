Multi Environment Static Website Deployment using CI/CD

Project Description

This project demonstrates automated deployment of a static website using Software Configuration Management and CI/CD pipeline. Whenever code changes are pushed to GitHub, the website automatically updates.

    Multi-Environment Static Website Deployment using CI/CD

    Introduction

    This project demonstrates Software Configuration Management (SCM) integrated with Continuous Integration and Continuous Deployment (CI/CD). The website automatically deploys whenever changes are pushed to GitHub.

    Problem Statement

    Manual deployment of websites leads to delays, configuration errors, and lack of version tracking. This project solves the issue by automating build, testing, and deployment using GitHub Actions.

    Project Objectives

    • Automate website deployment
    • Maintain version control
    • Enable team collaboration
    • Reduce deployment errors
    • Implement DevOps practices

    Software Configuration Management (SCM)

    SCM Tools Used

    • Git – Version Control
    • GitHub – Remote Repository
    • GitHub Actions – CI/CD Pipeline
    • GitHub Pages – Deployment Environment

    Configuration Items (CIs)

    • index.html – Website Structure
    • style.css – User Interface Design
    • script.js – Interactive Features
    • .github/workflows/deploy.yml – CI/CD Pipeline

    SCM Activities

    • Version Control using Git
    • Change Management through commits
    • Automated Build & Deployment
    • Configuration Auditing

    Project Advantages

    • Automatic website deployment
    • Continuous integration support
    • Error reduction
    • Faster delivery
    • Easy rollback to previous versions

    Project Uses

    • DevOps learning platform
    • Portfolio website deployment
